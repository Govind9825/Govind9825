### 📚 System Design Journey

```
Learning from first principles. Not just reading — understanding tradeoffs.

  Phase 1 — Core Foundations
  ✅  Authentication & Authorization   (JWT, OAuth 2.0, PKCE, Sessions, RBAC)
  ✅  REST API Design                  (Idempotency, Pagination, Versioning)
  ✅  Rate Limiting                    (Token Bucket, Sliding Window, Redis)
  ✅  Caching                          (LRU/LFU, Cache-Aside, Write-Through)

  Phase 2 — Databases (In Progress)
  🔄  SQL Deep Dive                    (Indexing, ACID, Transactions)
  🔄  Redis                            (Caching Patterns, Pub/Sub, Locks)
  ⏳  CAP Theorem & NoSQL Tradeoffs
  ⏳  Sharding & Replication

  Phase 3 — Distributed Systems (Upcoming)
  ⏳  Message Queues                   (Kafka, RabbitMQ)
  ⏳  Consistent Hashing & Load Balancers
  ⏳  Distributed Transactions         (2PC, Saga Pattern)

  Classic Problems to Design:
  ⏳  URL Shortener • Chat System • Payment System • Twitter Feed

  📖 Resource: System Design Interview — Alex Xu (ByteByteGo)
  🎯 Goal: Think like a senior engineer. Ask "what breaks at 1M users?"
```
